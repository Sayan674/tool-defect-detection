# Dataset Documentation

## Vision-Based Machining Tool Recognition and Health Assessment Framework

---

## Why a Custom Dataset?

No publicly available image dataset exists that covers the specific combination of machining tool classes — drill, reamer, milling cutter, gear cutter, lathe tool, and parting tool — captured under controlled workshop conditions.

Existing public datasets either:
- Focus on a single tool type (typically twist drills for wear progression studies)
- Contain images taken with specialised equipment (structured light, confocal microscopy) that is not available in a standard workshop
- Are not licensed for research reproduction

Building a custom dataset allowed full control over lighting, background, and image quality, which directly affects the reliability of the contour-based feature extraction pipeline.

## Camera Setup

| Parameter           | Specification                              |
|---------------------|--------------------------------------------|
| Camera              | Any smartphone camera ≥ 12 MP, or DSLR     |
| Distance to tool    | 20–30 cm                                   |
| Orientation         | Top-down (overhead) or front-facing flat   |
| Focus mode          | Auto-focus; ensure the tool is sharp       |
| Flash               | Disabled — use ambient / diffused lighting |

---

## Image Resolution

- **Minimum**: 640 × 640 px
- **Recommended**: 1200 × 1200 px or higher

Images are automatically rescaled to 500 × 500 px during preprocessing. Capturing at higher resolution gives the downsampling step more information to work with, resulting in cleaner silhouettes.

---

## Lighting Conditions

Lighting is the single biggest factor affecting segmentation quality.

✅ **Good:**
- Diffused natural light from a window (no direct sun)
- Overhead LED panel with a diffuser sheet
- Two desk lamps placed at 45° on either side of the tool (avoids harsh shadows)

❌ **Avoid:**
- Direct sunlight (creates hot-spots that confuse thresholding)
- Single-point light source (produces asymmetric shadows that distort contours)
- Fluorescent flicker (use shutter speed > 1/100 s to avoid banding)

---

## Background

The thresholding stage (`THRESH_BINARY_INV`, threshold = 60) assumes the tool is **darker than the background**. This is the most reliable segmentation strategy for metal tools under even lighting.

✅ **Recommended backgrounds:**
- Plain white A4 paper
- White foam board
- White cotton cloth (matte, no texture)

❌ **Avoid:**
- Patterned surfaces
- Reflective or glossy surfaces
- Backgrounds that are similar in brightness to the tool

## Image Naming Convention

Images must be placed in **per-class sub-folders** named exactly as the class labels listed above:

```
data/
└── raw_images/
    ├── drill/
    │   ├── drill_001.jpg
    │   ├── drill_002.jpg
    │   └── ...
    ├── reamer/
    ├── milling/
    ├── gear/
    ├── lathe/
    └── parting/
```

File names within each folder can be arbitrary, but a consistent `<class>_NNN.jpg` convention makes it easy to track individual images.

## Data Collection Methodology

1. **Place the tool on a plain white background** on a flat surface.
2. **Position the camera directly overhead** (top-down view), centred above the tool. For elongated tools (parting blade, drill), orient the tool horizontally in frame.
3. **Fill ~60% of the frame** with the tool — not too close (risk of missing the full profile), not too far (loss of detail on features like gear teeth).
4. **Check focus**: tap on the tool on the smartphone screen to lock focus.
5. **Capture 3–5 images** per physical positioning; vary the tool orientation slightly (±10°) between shots to increase diversity.
6. **Review each image** before moving on: the tool should be clearly separated from the background with no shadows crossing the tool profile.
7. **Discard blurry or poorly lit images** before running extraction.

## Dataset Recreation Steps

```bash
# 1. Capture images following the guidelines above and place them in:
#    data/raw_images/<class_name>/

# 2. Extract features from all images
python main.py --step extract

# 3. Generate synthetic augmentation and build the training CSV
python main.py --step augment

# 4. Train the classifier
python main.py --step train

# 5. Evaluate
python main.py --step evaluate
```

## Best Practices

- Capture images of **both new and used tools** to give the classifier exposure to geometric variation.
- Avoid capturing multiple tools in a single image — the pipeline selects the largest contour; extra tools in frame will confuse it.
- Re-check the threshold value (`THRESHOLD_VALUE` in `config.py`) if your background is not pure white. Lowering it may help on grey or beige backgrounds.
- For very shiny tools (HSS drills, carbide inserts), placing a thin matte paper between the tool and the light source can reduce specular highlights.
- Check the extracted silhouettes by running the preprocessing visually on a few sample images before committing to a full extraction run.
